# Project Summary & Innovation
**แอปพลิเคชัน AI Finance Coach "พี่เงิน"**

---

## 1. สรุปปัญหา (Pain Points) และโซลูชัน (Solutions)

**ปัญหาที่พบ (Pain Points):**
จากการเก็บข้อมูลกลุ่มเป้าหมาย (นักศึกษาและวัยทำงาน) พบปัญหาหลัก 3 ประการในการจัดการการเงิน:
1. **ความยุ่งยากในการบันทึกรายรับ-รายจ่าย:** การต้องพิมพ์ตัวเลขและข้อมูลเองทุกครั้งทำให้เกิดความขี้เกียจ และนำไปสู่การล้มเลิกการทำบัญชีกลางคัน
2. **ไม่เห็นภาพรวมของเงินตัวเอง:** หลายคนใช้เงินเดือนชนเดือน ไม่ทราบว่าเงินหายไปกับหมวดหมู่ใดมากที่สุด และไม่รู้วิธีควบคุมงบประมาณ
3. **ขาดความรู้และที่ปรึกษาด้านการเงิน:** การจ้างที่ปรึกษาทางการเงินมีค่าใช้จ่ายสูง ส่วนการหาข้อมูลเองก็มักจะได้คำแนะนำกว้างๆ ที่ไม่ตรงกับบริบททางการเงินของตนเอง

**โซลูชันที่นำเสนอ (Solutions):**
แอปพลิเคชัน "พี่เงิน" ถูกสร้างขึ้นมาเพื่อแก้ปัญหาเหล่านี้ โดยผสมผสานเครื่องมือจัดการการเงินเข้ากับเทคโนโลยี AI สมัยใหม่:
- มีระบบบันทึกรายจ่ายอัตโนมัติผ่านการสแกนภาพ (แก้ปัญหาขี้เกียจพิมพ์)
- มี Dashboard อัจฉริยะที่สรุปยอดและมีระบบแจ้งเตือนเมื่อใกล้ใช้เกินงบ (แก้ปัญหาเงินชนเดือน)
- มี "พี่เงิน" AI Chatbot ที่คอยตอบคำถามและวางแผนการเงินให้ส่วนบุคคลตลอด 24 ชม. (แก้ปัญหาขาดที่ปรึกษา)

## 2. นวัตกรรมและเทคโนโลยีที่นำมาแก้ปัญหา (Innovation)

จุดเด่นของแอปพลิเคชันนี้ไม่ได้เป็นเพียงแค่แอปบันทึกบัญชีธรรมดา แต่มีการยกระดับประสบการณ์ผู้ใช้งานด้วยนวัตกรรม 2 ส่วนหลัก ดังนี้:

### 2.1 เทคโนโลยี OCR Extract & Auto-Categorization
* **นวัตกรรม:** การใช้ **Typhoon OCR (server-side)** ทำหน้าที่วิเคราะห์ภาพสลิปโอนเงินธนาคาร — รูปถูกส่งไปประมวลผลที่ backend (`/transactions/parse-slip`) แล้วสกัดข้อมูลด้วย `parser.ts` ทำให้ไม่ต้องพึ่งไลบรารีบนเครื่อง (on-device) และปรับปรุงความแม่นยำได้จากฝั่งเซิร์ฟเวอร์
* **ผลลัพธ์:** ผู้ใช้งานเพียงแค่อัปโหลดหรือถ่ายรูปสลิปโอนเงิน ระบบจะดึง "จำนวนเงิน", "วันที่", และ "ชื่อผู้รับ/บันทึกช่วยจำ" ออกมาอัตโนมัติ (ไม่ดึงรหัสอ้างอิงมาปน) ยิ่งไปกว่านั้นระบบยังจัดหมวดหมู่ (Auto-Categorize) ให้เองจากคีย์เวิร์ด ช่วยลด Friction ในการใช้งานได้มหาศาล ทำให้ผู้ใช้มีวินัยในการบันทึกบัญชีได้ต่อเนื่อง

### 2.2 Context-Aware AI Finance Coach (Typhoon LLM)
* **นวัตกรรม:** หลายแอปพลิเคชันที่มี AI มักจะเป็นเพียงการเชื่อมต่อ API ของ ChatGPT ธรรมดา (Basic Wrapper) แต่ "พี่เงิน" ใช้เทคนิค **Structured Context Injection** (ประกอบสรุปข้อมูลการเงินของผู้ใช้เป็น context แนบไปกับ prompt — ต่างจาก RAG ตรงที่ไม่ต้องมี vector store) ร่วมกับ **Typhoon LLM** (AI ที่เก่งภาษาไทยและเข้าใจบริบทคนไทยที่สุด) โดยมี **provider fallback** (Groq/OpenAI/rule-based) กันล่ม
* **ผลลัพธ์:** เมื่อผู้ใช้งานแชทปรึกษา ระบบ (Backend) จะทำการสรุปพฤติกรรมการใช้จ่ายของผู้ใช้ในช่วง 30 วันที่ผ่านมา (เช่น รายได้, ยอดหนี้, หมวดหมู่ที่ใช้เงินมากที่สุด, สถานะเป้าหมายการออม) แล้วแนบข้อมูลนี้ไปเป็น Context เบื้องหลังให้กับ LLM
* **คุณค่าที่ได้รับ:** ทำให้ "พี่เงิน" สามารถให้คำปรึกษาที่ **"เฉพาะเจาะจงกับสภาพการเงินของบุคคลนั้นจริงๆ"** (Hyper-Personalized) ไม่ใช่แค่การตอบคำถามกว้างๆ แบบหุ่นยนต์ เช่น หากผู้ใช้ถามว่า "เดือนนี้ควรซื้อรองเท้าคู่ใหม่ดีไหม?" พี่เงินจะสามารถตอบได้ว่า "ตอนนี้งบช้อปปิ้งของคุณเหลือเพียง 500 บาท หากซื้ออาจจะกระทบเป้าหมายการออมไปเที่ยวญี่ปุ่นเดือนหน้านะครับ"
